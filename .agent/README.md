# .agent Directory Guide

這個目錄是用來存放與 AI 協作相關的設定與文件。除了 Workflows，你還可以放置以下幾種內容：

## 1. Workflows (`.agent/workflows/`)
標準化的工作流程腳本，定義特定任務的執行步驟。
- **格式**：Markdown 檔案 (.md)
- **用途**：重複性高、步驟明確的任務 (例如：部署、建立新模組)。

## 2. Skills (技能) (`.agent/skills/`)
教導 AI 完成複雜任務的專屬技能包。
- **結構**：每個技能一個資料夾，內含 `SKILL.md`。
- **範例**：
    ```text
    .agent/skills/refactor-component/
    └── SKILL.md  (定義重構組件的詳細邏輯與規則)
    ```
- **用途**：當任務邏輯複雜，無法透過簡單步驟描述時使用。

## 3. Rules / Context (規則與背景) (`.agent/rules.md` 或其他)
專案層級的開發規範與背景知識，供 AI 隨時參考。
- **用途**：
    - **Coding Style**：縮排、命名慣例、禁止使用的語法。
    - **架構決策**：為什麼選用這個套件、專案目錄結構意義。
    - **業務邏輯**：核心業務規則說明。
- **範例**：目前的 `rules.md` 就是這類文件。

## 4. Knowledge (知識庫) (`.agent/knowledge/` - 非強制，慣例)
存放與專案相關的技術文件摘要或備忘錄，讓 AI 快速查找。
- **用途**：API 規格書摘要、資料庫 Schema 說明。

---

**小撇步**：
只要將重要資訊放在這個資料夾下，AI 在執行任務時就有機會參考到這些內容，協助你維持專案的一致性。
